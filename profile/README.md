# The Camus Project

The Camus Project is an open initiative building a unified method, specification language, programming language, and tooling for software generated by AI and assumed by humans.

Camus aims to make software systems:

- explicit rather than implicit,
- verifiable rather than opaque,
- maintainable rather than disposable,
- understandable by humans and machines alike.

The project explores a different approach to software engineering for an era where code generation is becoming abundant, but human responsibility remains unavoidable.

## Core Components

- **Camus Method**  
  An engineering method centered on explicit intent, traceability, and progressive verification.

- **Camus SL**  
  A specification language designed to express intent in a form understandable by both humans and AI systems.

- **Camus PL**  
  A programming language focused on readability, explicit semantics, and long-term maintainability.

- **Camus Tooling**  
  Supporting tools for validation, analysis, generation, traceability, and repository integration.

## Principles

- Open by design
- Human-assumable systems
- Explicit intent over implicit behavior
- Long-term maintainability
- AI-assisted, not AI-obscured
- Progressive trust and verification

---

> Software generated by AI must still be understood, verified, and assumed by humans.

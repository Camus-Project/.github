# The Camus Project

> Misname an object adds to the misfortune of this world.
> 
> Mal nommer un objet, c’est ajouter au malheur de ce monde.
> 
> — Albert Camus, *Sur une philosophie de l’expression*, 1944

The Camus Project builds a unified method, specification language, programming language, and tooling for software generated by AI and assumed by humans. The project develops its methods and tooling by applying them to itself — a self-hosted approach to method engineering.

Camus describes software systems along four dimensions:

- explicit rather than implicit,
- verifiable rather than opaque,
- maintainable rather than disposable,
- understandable by humans and machines alike.

The project addresses an era where code generation is abundant, but human responsibility remains unavoidable.

## Core Components

- **Camus Method**  
  An engineering method centered on explicit intent, traceability, and progressive verification.

- **Camus SL**  
  A specification language that expresses intent in a form understandable by humans and AI systems.

- **Camus PL**  
  A programming language for readability, explicit semantics, and long-term maintainability.

- **Camus Tooling**  
  Tools for validation, analysis, generation, traceability, and repository integration.

- **Camus Skill**  
  An embeddable skill that brings the Camus Method and SL into AI-assisted software engineering workflows.  
  [`camus-skill` repo](https://github.com/Camus-Project/camus-skill)

## Principles

The project operates under the following structural constraints:

- Open by design
- Systems remain human-assumable
- Explicit intent over implicit behavior
- Long-term maintainability
- AI-assisted generation, human verification
- Progressive trust through explicit certification

---

> Software generated by AI must still be understood, verified, and assumed by humans.
